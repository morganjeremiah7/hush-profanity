# Multi-word phrases the matcher will treat as a single profanity hit.
# One phrase per line, lowercase, words separated by single spaces.
# Lines starting with # are comments.
#
# These exist because Whisper sometimes splits compounds ("bullshit" -> "bull shit"),
# and because some expletives are inherently multi-word ("oh my god"). Adding the
# single words to swears.txt would over-fire (e.g. "god" alone fires too often).
#
# A phrase always wins over single-word matches if both apply at the same position
# (greedy longest-match). The matcher tries phrase variants with and without
# internal apostrophes, so "for god's sake" also catches "for gods sake".
#
# The cleaned-subtitle replacement for these comes from the "_phrases" section of
# config/replacements.json. Anything without a specific replacement uses _default.
